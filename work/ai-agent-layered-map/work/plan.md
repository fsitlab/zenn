# 作業計画書

## 概要
複数のMDファイルを統合し、新しい5層モデルに基づいた記事を作成する。

## 新しい階層構造

```
第1層：LLM層
  - ファインチューニング、LoRA
  - モデル系列（テキスト/画像/動画対応範囲）
  - 読める/書けるの違い
  - モデルごとのチューニング（Claude憲法、Googleマルチモーダル、Anthropic AGI等）

第2層：通信層
  - 2層A：クライアント起点の通信（プロンプトチューニング、ツール一覧宣言）
  - 2層B：LLM起点の通信（ツール利用判断）
  - OpenAI API、Anthropic API、SSE対応など

第3層：アプリ層
  - LangChain等プログラム系
  - Dify等ローコード系
  - ChatGPT等コンシューマアプリ
  - Claude Code等コーディングエージェント
  - Perplexity、AWS Strands Agents、AgentCore
  - 従来型RAG、ワークフロー

第4層：外部ツール層
  - RAGのベクトルDB
  - リモートMCPサーバー
  - サブエージェント
  - ローカルファイル読み書きプログラム
  - 位置分類：a:ローカル環境、b:サンドボックス環境、c:クラウド環境
  - 内容分類：α:ファイル集合体、β:プログラム、γ:AI
  - 影響分類：CRUD（特にUDは権限制御重要）

第5層：プレゼンテーション層
  - Antigravity、VSCode等UI専用サービス
  - Difyワークフロー（ローコード）
```

## 出力ファイル構成

```
./output/
├── 00_はじめに.md          # 概要・目的・5層モデル全体図
├── 01_LLM層.md             # 第1層の詳細
├── 02_通信層.md            # 第2層（A/B含む）の詳細
├── 03_アプリ層.md          # 第3層の詳細
├── 04_外部ツール層.md      # 第4層の詳細
├── 05_プレゼンテーション層.md  # 第5層の詳細
├── 06_早見表.md            # 技術と層の対応表
├── 07_Claude_Cowork.md     # Claude Coworkの仕組み調査
├── 08_相性問題.md          # 相性問題の整理
└── 09_付録.md              # ディレクトリ構造比較、リンク集等
```
